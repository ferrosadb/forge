## Test-case generation rule

Every RPN ≥ 50 row MUST have a corresponding test referenced in `test-plan.md`. CI audits by grep of `F\d+` IDs in test names/docstrings. CRITICAL rows (RPN ≥ 200) additionally require an integration or fixture test (not just unit) — the mitigation is worthless without end-to-end proof.
