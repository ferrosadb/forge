# forge-dsm-analyze

Design Structure Matrix (DSM) analysis and dead-code detection for `frg`.

## What's here

- `extract/` — regex-based, per-language extractors for dependency edges
  (`Extractor`) and symbol declarations/references (`DeclarationExtractor`).
  Languages: Rust, Python, Go, TypeScript, Elixir, Java, C#, plus
  cross-language FFI/IPC bridges.
- `matrix`, `cycles`, `cluster`, `partition`, `metrics`, `suggest`, `report`,
  `directed`, `enforce` — the DSM pipeline behind `frg dsm analyze`.
- `dead_code` — BFS reachability from entry points over the extracted
  declaration/reference graph (`frg dsm dead-code`, MCP tool `dead_code`).

## Dead-code model

`find_dead_code` builds a file-granularity reference graph: a reference found
in a file creates edges from every symbol declared in that file to the
referenced declaration (resolved by `::`-suffix match), plus implicit
member → module edges so a module is alive whenever any declaration under its
fully-qualified name is alive (e.g. a `mod operator;` holding only `impl`
blocks). BFS runs from entry points (`main`, `#[test]`/`#[tokio::main]`
attributes, `#[no_mangle]`, `#[wasm_bindgen]`, constructors named `new`, and
linker/runtime-retained statics: `#[global_allocator]`, `#[used]`,
`#[link_section]`, `#[panic_handler]`). Unreachable declarations are reported
as:

- **Definite** — private/internal symbols unreachable from any entry point.
- **Possible** — public symbols with no internal references (may be consumed
  by external crates). This boundary is intentionally conservative; do not
  promote Possible to Definite.

### Filters applied before analysis

- **Generated files** are skipped when one of these markers appears in the
  first 20 lines: `@generated`, `DO NOT EDIT`, `Generated by`,
  `generated by`, `Code generated` (wit-bindgen, prost-build, protoc, …).
- **Exclude globs** (`--exclude`, repeatable; MCP param `exclude`) match the
  path relative to the scanned root. `**` crosses segments, `*` stays within
  a segment, a bare file name matches at any depth. Invalid patterns fail
  loud.
- **Test code**: Rust declarations under `#[cfg(test)]` (attribute on the
  item or an enclosing `mod`, including out-of-line `#[cfg(test)] mod foo;`
  whose contents live in `foo.rs`) are marked `is_test` and excluded unless
  `--include-tests` is set, in addition to the older path/name heuristics
  (`/tests/`, `_test.`, `test_`, `::tests::`).

### Reference patterns captured (Rust)

Qualified paths (`foo::bar`), left-hand path segments (`commands::run()`,
`use m::{a, b}` reference module `commands`/`m`), turbofish calls
(`data_array::<16>(…)`), function calls (`foo(`), method calls (`.foo(`),
type annotations, struct construction, generics, `impl` targets,
`#[derive(...)]` traits, string-path attributes (`#[serde(with = "mod")]`),
format-string interpolations (`format!("{prefix}/{MANIFEST_PATH}")`), and
**bare SCREAMING_SNAKE_CASE identifiers** in expressions (`FOO.inc()`,
`x = FOO + 1`, `&FOO`). Bare-identifier, turbofish, and path-segment
scanning run on comment- and string-stripped lines and skip the declaring
line, so a constant mentioned only in a doc comment or log string still
counts as dead.

Associated `type` items inside `impl` blocks (`impl Deref for S { type
Target = …; }`) are not extracted as declarations — they are members of the
impl, not standalone auditable symbols.

## Known imprecision (regex-based extraction — read before trusting output)

The extractors deliberately trade precision for zero build/toolchain
dependencies. Findings are *candidates*, not proof. Known gaps:

- **Trait-impl method dispatch**: a method invoked only through a trait
  object or generic bound may be resolved by name-suffix only; distinct
  methods with the same name are conflated (can hide dead code and can mark
  dead methods live).
- **Macro bodies**: symbols used only inside `macro_rules!` expansions or
  attribute-macro-generated code are not seen; such symbols can be falsely
  reported dead.
- **Bare-identifier references** are limited to SCREAMING_SNAKE_CASE
  (constant/static naming convention). Non-conventional lowercase constants
  used only as bare identifiers can still be falsely flagged.
- **Comment/string stripping** is line-oriented: raw strings (`r#"…"#`),
  multi-line string literals, and quote characters in char literals are
  handled approximately. A screaming-case word inside such a literal may
  still count as a reference (suppressing a finding, never inventing one).
- **`#[cfg(test)]` tracking** uses brace counting; braces inside strings or
  comments can extend the detected test region (again: suppresses findings
  rather than inventing them).
- **File-granularity edges**: any reference in a file makes the target
  reachable from *all* symbols declared in that file, which over-approximates
  reachability (fewer findings, no extra false positives).
- **Re-exports / `pub use` facades** count as references from the exporting
  file; the true external consumer is not tracked.
- **Left-hand path segments** resolve by suffix, so two same-named private
  modules in different crates can keep each other alive (suppresses
  findings).
- **`#[allow(dead_code)]` is not consulted**: a deliberately retained but
  unreferenced item is still reported — the report states reference facts,
  not intent.

Where a gap can *invent* a false "definitely dead" finding, prefer fixing the
extractor; where it merely suppresses a finding, it is documented here and
accepted.
